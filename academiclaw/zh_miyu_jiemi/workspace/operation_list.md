# 谜语解谜任务操作参考

## 操作步骤

### 1. 理解任务要求
- 阅读 Query.md 文件了解详细要求
- 分析 query.json 中的谜语数据
- 了解 submission.json 的格式要求
- 确定任务目标和方向

### 2. 数据准备
- 读取 query.json 文件获取谜语列表
- 理解每个谜语的 id、category 和 riddle 字段
- 准备生成预测结果的数据结构

### 3. 谜语解答
- 分析每个谜语的含义和线索
- 运用中文知识和推理能力
- 生成每个谜语的预测答案
- 确保答案简洁准确

### 4. 结果生成
- 创建 submission.json 文件
- 按照指定格式组织预测结果
- 确保包含所有谜语的预测
- 验证 JSON 格式正确性

### 5. 测试和验证
- 运行评分脚本验证结果
- 检查准确率是否达到要求
- 分析错误和遗漏的情况
- 优化预测算法

### 6. 文档和提交
- 确保所有文件格式正确
- 验证提交文件完整性
- 准备最终提交
- 检查评分结果

## 注意事项
- 确保网络连接稳定
- 预留足够的思考和推理时间
- 验证所有输出文件的格式
- 避免修改现有文件
- 确保预测结果简洁准确
- 不要包含解释，只包含最终答案

## 参考工具
- JSON 处理：json 库
- 文件操作：os, shutil
- 评分验证：rubric/rubric_riddles.py
- 数据分析：pandas, numpy

## 常见问题与解决方案
- JSON 格式错误：检查 JSON 语法和结构
- 预测不准确：深入分析谜语含义
- 缺少预测：确保覆盖所有谜语
- 评分失败：检查文件路径和格式
- 答案过长：确保只包含简短答案
